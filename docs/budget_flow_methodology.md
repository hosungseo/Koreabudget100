# 세부사업 예산체계도 생성 방법

## 1. 목적

예산체계도의 기본 단위는 **세부사업 한 개**다. 조직의 업무 순서를 그리는 것이 아니라, 국회가 확정한 돈이 어떤 사업 내용과 회계 세목으로 나뉘고 어떤 집행채널과 기관·지역·수혜 근거로 이어지는지를 보여 준다.

단일 사업 주 화면의 기본 읽기 구조는 다음과 같다.

```text
중앙 확정재원 → ├ PDF 내용별 분해 ─┐
                    └ Add2 회계별 분해 ─┴ 내용·금액 교차 대사
                                             └ 지방이전 분기 → LOFIN 명칭 후보·시계열
```

범용 탐색기의 G0~G5는 법정 절차 단계가 아니라 돈의 위치를 비교하기 위한 표시 계층이다. 단일 주 화면은 내용 분해와 회계 분해가 순차 흐름으로 오인되지 않도록 G단계 배치를 사용하지 않는다.

| 계층 | 의미 |
|---|---|
| G0 중앙 확정재원 | 회계·분야·부문과 세부사업 국회확정액 |
| G1 내역사업 | 부처 설명자료의 사업별 산출 배분 |
| G2 목·세목·채널 | 열린재정 Add2 회계 세목과 집행유형 |
| G3 중앙 집행대상 | 세목·문서로 확인된 수급 유형·기관과 미상 구분 |
| G4 지방 편성 후보 | LOFIN 지방 세부사업의 예산현액과 국비·시도비·시군구비·기타 재원 |
| G5 지출·수혜·검증 | LOFIN 지출 스냅샷, 수혜자 근거, API·PDF 합계 대사 |

## 2. 두 개의 예산 분해를 섞지 않는다

한 세부사업 총액은 서로 다른 두 방향으로 분해된다.

1. **사업 내용 분해**: PDF의 내역사업과 산출근거
2. **회계 분해**: 열린재정 Add2의 목·세목

둘은 합계가 같아도 항목 간 일대일 대응표가 아니다. 따라서 각각 세부사업 국회확정액과 원 단위로 대사한다. 내역사업↔세목 연결은 문서 표현, 지원율, 금액이 함께 맞는 부분만 별도 대사선으로 표시한다. 이름이 비슷하다는 이유로 나머지를 자동 연결하지 않는다.

## 3. 자료별 역할

### 열린재정 `ExpenditureBudgetAdd2`

- 연도, 부처, 회계, 분야, 부문, 프로그램, 단위사업, 세부사업의 정본
- 국회확정액 `Y_YY_DFN_KCUR_AMT`의 정본
- 목·세목 금액과 비중
- 세목 명칭에 따른 집행채널 분류

### 부처 세부사업 설명자료 PDF

- 내역사업명, 2026년 예산, 산출 수량과 단가
- 시행방법, 시행주체, 수혜자
- 보조·융자·출연·출자 지원금액과 지원율
- 문서 파일, 페이지, Kordoc 청크 근거

PDF는 열린재정 총액을 덮어쓰지 않는다. 확정 매칭된 설명자료만 붙이고 `ambiguous`와 `unmatched`는 정규 사업에 자동 부착하지 않는다.

### 지방재정365 `QWGJK`

- 중앙 세부사업명 또는 PDF에 명시된 내역사업명으로 탐색한 지방 세부사업 후보
- 지자체명·코드, 권역, 광역/기초 구분, 지방 세부사업명·코드
- 예산현액, 국비·시도비·시군구비·기타, 편성액, 지출액, 관측 집행률
- 조회 기준일, 검색 키워드, 연결 범위(`central_business_keyword` 또는 `pdf_subproject_keyword`)

현재 연결은 `keyword_candidate`이며 교부·수령 확정 관계가 아니다. `exe_ymd`는 개별 거래일이 아니라 해당 날짜까지의 지방 세부사업 예산·지출 조회 기준일로 해석한다. 광역·기초 행은 같은 재원을 중복 표현할 수 있으므로 중앙예산에 가산하지 않고, 선 굵기나 합계 대사에도 사용하지 않는다. 관측 비율은 각 LOFIN 행 또는 후보군의 `지출액 ÷ 예산현액`으로만 계산하며 중앙정부 집행률로 해석하지 않는다.

## 4. 집행채널 분류

목·세목은 다음 규칙으로 한 채널에만 배치한다.

| 채널 | 대표 세목 신호 |
|---|---|
| 지방자치단체 보조 | 자치단체경상보조, 자치단체자본보조 |
| 민간위탁 | 민간위탁사업비 |
| 민간보조 | 민간경상보조, 민간자본보조 |
| 출연 | 기관운영출연금, 사업출연금 등 출연 세목 |
| 용역·연구계약 | 일반용역비, 관리용역비, 정책·일반연구비 |
| 융자·출자 | 융자·대여·출자 세목 |
| 시설·자산 | 시설비, 건설, 자산취득비 |
| 인건비 | 보수·인건비 계열 |
| 운영·사업관리 | 수용비, 여비, 임차료, 업무추진비 등 |
| 기타 집행 | 명칭만으로 위 유형을 확정할 수 없는 세목 |

채널 분류는 **수급기관의 실제 확정 증거가 아니다**. PDF가 기관을 명시할 때만 기관명을 붙이며, 그렇지 않으면 `기관 미상`으로 남긴다.

## 5. 기준 예시의 대사

기본 화면은 `지역사회 자생적 창조역량 강화` 사업이다.

- 국회확정액: 23,753백만원
- PDF 내역사업 5개 합계: 23,753백만원
- Add2 목·세목 8개 합계: 23,753백만원
- API 대사 차이: 0원
- PDF 내역사업 대사 차이: 0원
- 자치단체경상보조: 11,950백만원
- PDF에 보조율 50%가 명시된 두 항목: 1,600 + 10,350 = 11,950백만원
- 범용 자동 대사에서 문서·지원율·금액으로 직접 연결한 금액: 14,421백만원
- 단일 사례에서 PDF 산출내용과 금액을 수작업 검토한 교차 대사: 23,753백만원

| PDF 내역사업 | Add2 회계버킷 배분 | 합계 |
|---|---|---:|
| 청년 유입·체류 | 일반용역 66.89억 + 관리 0.75억 | 67.64억 |
| 데이터 기반 | 민간위탁 9.82억 + 관리 0.18억 | 10.00억 |
| 민관협력체계 | 정책연구 3.00억 + 일반용역 21.50억 + 관리 0.50억 | 25.00억 |
| 지역역량성장거점 | 지자체보조 16.00억 + 관리 0.50억 | 16.50억 |
| 사회연대경제 활성화 | 지자체보조 103.50억 + 일반용역 11.00억 + 관리 3.89억 | 118.39억 |
| **회계버킷 합계** | **지자체보조 119.50 + 일반용역 99.39 + 민간위탁 9.82 + 정책연구 3.00 + 관리 5.82억** | **237.53억** |

이 교차 대사는 실제 자금이 세목 사이를 순차적으로 이동했다는 뜻이 아니다. PDF 산출내용을 Add2 회계 분류와 대조한 **내용·금액 기반 검토**이다.

`17개 광역시도 전수조사 300백만원`은 조사의 **대상 범위**다. 광역시도가 그 돈의 수령자라는 근거가 아니므로 용역·연구 수행기관과 구분한다.

반면 PDF 내역사업 `사회연대경제 활성화`는 118.39억원 전체가 지방이전이 아니다. 지방 후보 연결 기준은 문서에 보조율 50%로 명시된 자치단체경상보조 103.50억원이고, 나머지는 일반용역 11.00억원과 사업관리 3.89억원이다.

QWGJK에서 키워드 `사회연대경제`를 `2026-01-31`, `02-28`, `03-31`, `04-30`, `05-31`, `06-30`, `07-18`에 반복 조회한다. 최신 국비 양수 27행은 다음 근거 등급으로 분리한다.

| 등급 | 판정 기준 | 2026-07-18 |
|---|---|---:|
| A 강한 후보 | PDF의 특이 표현 `혁신모델`을 포함 | 9행 |
| B 포괄 유사 | `생태계 활성화`, `활력제고`, `활성화(보조)` | 3행 |
| C 별도 사업 가능성 | PDF에 없는 `청년 일경험` 계열 | 15행 |

A 후보 역시 국비 100%로 표시되어 PDF 보조율 50%와 맞지 않으므로 확정 수혜지가 아니다. 27행 전체의 국비 관측합 155.36956억원은 중앙 보조액 103.50억원보다 51.86956억원 크므로 확정 배분표로 쓸 수 없다. 시계열의 행 수·예산현액·지출액 변화도 교부 이력이 아니라 매 조회일의 스냅샷 변화로만 읽는다.

## 6. 품질 원칙

- 모든 세부사업의 목·세목 합계와 채널 합계는 국회확정액과 같아야 한다.
- 국회확정액이 0원인 사업은 목·세목과 집행채널을 임의 생성하지 않는다.
- PDF 내역사업 합계가 총액과 다르면 차이를 경고하고 원문 항목을 보존한다.
- 범용 탐색기는 내역사업↔세목 직접 대사율이 낮아도 자동 추정으로 채우지 않는다. 단일 전액 대사는 산출내용·금액을 사람이 검토한 별도 근거 등급으로 표시한다.
- 기관·지역·수혜자는 자료에 명시된 범위까지만 표시한다.
- 각 PDF 사실에는 파일, 페이지, 청크 범위를 남긴다.
- LOFIN 후보는 항상 비가산·후보 상태로 표시한다.
- 인쇄와 모바일 화면에서도 정본, 문서 근거, 분류, 후보, 미상 상태를 텍스트로 구분한다.

핵심 원칙은 **돈의 구조는 더 자세히 보이게 하되, 자료가 말하지 않은 수급 관계는 만들지 않는 것**이다.
