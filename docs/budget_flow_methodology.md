# 세부사업 예산체계도 생성 방법

## 1. 목적

예산체계도의 기본 단위는 **세부사업 한 개**다. 조직의 업무 순서를 그리는 것이 아니라, 국회가 확정한 돈이 어떤 사업 내용과 회계 세목으로 나뉘고 어떤 집행채널과 기관·지역·수혜 근거로 이어지는지를 보여 준다.

기본 읽기 순서는 다음과 같다.

```text
재원·회계 → 세부사업·내역 → 목·세목 → 집행채널 → 기관·지역·수혜
```

화면의 G0~G4는 법정 절차 단계가 아니라 돈의 위치를 비교하기 위한 표시 계층이다.

| 계층 | 의미 |
|---|---|
| G0 확정재원 | 회계·분야·부문과 세부사업 국회확정액 |
| G1 내역사업 | 부처 설명자료의 사업별 산출 배분 |
| G2 목·세목·채널 | 열린재정 Add2 회계 세목과 집행유형 |
| G3 기관·지역 | 자료로 확인된 수급 유형·기관·지역 후보와 미상 구분 |
| G4 수혜·검증 | 수혜자 근거, API 합계 대사, PDF 합계 대사 |

## 2. 두 개의 예산 분해를 섞지 않는다

한 세부사업 총액은 서로 다른 두 방향으로 분해된다.

1. **사업 내용 분해**: PDF의 내역사업과 산출근거
2. **회계 분해**: 열린재정 Add2의 목·세목

둘은 합계가 같아도 항목 간 일대일 대응표가 아니다. 따라서 각각 세부사업 국회확정액과 원 단위로 대사한다. 내역사업↔세목 연결은 문서 표현, 지원율, 금액이 함께 맞는 부분만 별도 대사선으로 표시한다. 이름이 비슷하다는 이유로 나머지를 자동 연결하지 않는다.

## 3. 자료별 역할

### 열린재정 `ExpenditureBudgetAdd2`

- 연도, 부처, 회계, 분야, 부문, 프로그램, 단위사업, 세부사업의 정본
- 국회확정액 `Y_YY_DFN_KCUR_AMT`의 정본
- 목·세목 금액과 비중
- 세목 명칭에 따른 집행채널 분류

### 부처 세부사업 설명자료 PDF

- 내역사업명, 2026년 예산, 산출 수량과 단가
- 시행방법, 시행주체, 수혜자
- 보조·융자·출연·출자 지원금액과 지원율
- 문서 파일, 페이지, Kordoc 청크 근거

PDF는 열린재정 총액을 덮어쓰지 않는다. 확정 매칭된 설명자료만 붙이고 `ambiguous`와 `unmatched`는 정규 사업에 자동 부착하지 않는다.

### 지방재정365 `QWGJK`

- 중앙사업명으로 탐색한 지방 세부사업 후보
- 지자체명, 광역/기초 구분, 국비·시도비·시군구비
- 조회 기준일과 검색 키워드

현재 연결은 `keyword_candidate`이며 교부·수령 확정 관계가 아니다. 광역·기초 행은 같은 재원을 중복 표현할 수 있으므로 중앙예산에 가산하지 않고, 선 굵기나 합계 대사에도 사용하지 않는다.

## 4. 집행채널 분류

목·세목은 다음 규칙으로 한 채널에만 배치한다.

| 채널 | 대표 세목 신호 |
|---|---|
| 지방자치단체 보조 | 자치단체경상보조, 자치단체자본보조 |
| 민간위탁 | 민간위탁사업비 |
| 민간보조 | 민간경상보조, 민간자본보조 |
| 출연 | 기관운영출연금, 사업출연금 등 출연 세목 |
| 용역·연구계약 | 일반용역비, 관리용역비, 정책·일반연구비 |
| 융자·출자 | 융자·대여·출자 세목 |
| 시설·자산 | 시설비, 건설, 자산취득비 |
| 인건비 | 보수·인건비 계열 |
| 운영·사업관리 | 수용비, 여비, 임차료, 업무추진비 등 |
| 기타 집행 | 명칭만으로 위 유형을 확정할 수 없는 세목 |

채널 분류는 **수급기관의 실제 확정 증거가 아니다**. PDF가 기관을 명시할 때만 기관명을 붙이며, 그렇지 않으면 `기관 미상`으로 남긴다.

## 5. 기준 예시의 대사

기본 화면은 `지역사회 자생적 창조역량 강화` 사업이다.

- 국회확정액: 23,753백만원
- PDF 내역사업 5개 합계: 23,753백만원
- Add2 목·세목 8개 합계: 23,753백만원
- API 대사 차이: 0원
- PDF 내역사업 대사 차이: 0원
- 자치단체경상보조: 11,950백만원
- PDF에 보조율 50%가 명시된 두 항목: 1,600 + 10,350 = 11,950백만원
- 문서와 금액으로 내역↔세목 직접 연결한 금액: 14,421백만원

`17개 광역시도 전수조사 300백만원`은 조사의 **대상 범위**다. 광역시도가 그 돈의 수령자라는 근거가 아니므로 용역·연구 수행기관과 구분한다. LOFIN에서 이 사업의 개별 지자체 수령 내역을 확인하지 못했으므로 지자체명은 만들지 않는다.

## 6. 품질 원칙

- 모든 세부사업의 목·세목 합계와 채널 합계는 국회확정액과 같아야 한다.
- 국회확정액이 0원인 사업은 목·세목과 집행채널을 임의 생성하지 않는다.
- PDF 내역사업 합계가 총액과 다르면 차이를 경고하고 원문 항목을 보존한다.
- 내역사업↔세목 직접 대사율이 낮아도 자동 추정으로 채우지 않는다.
- 기관·지역·수혜자는 자료에 명시된 범위까지만 표시한다.
- 각 PDF 사실에는 파일, 페이지, 청크 범위를 남긴다.
- LOFIN 후보는 항상 비가산·후보 상태로 표시한다.
- 인쇄와 모바일 화면에서도 정본, 문서 근거, 분류, 후보, 미상 상태를 텍스트로 구분한다.

핵심 원칙은 **돈의 구조는 더 자세히 보이게 하되, 자료가 말하지 않은 수급 관계는 만들지 않는 것**이다.
