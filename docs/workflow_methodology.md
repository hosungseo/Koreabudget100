# 세부사업 업무 절차 보조 화면 생성 방법

## 1. 목적

이 문서는 주 화면인 예산체계도가 아니라, PDF에 명시된 업무 순서를 확인하는 **보조 화면**의 생성 원칙을 설명한다. 예산의 출처·내역사업·목·세목·집행채널·기관·지역·수혜 구조는 `docs/budget_flow_methodology.md`와 `artifacts/budget_flow_map.html`을 따른다.

업무 절차 보조 화면은 **누가, 어떤 근거로, 어떤 순서로 사업을 수행하는지**를 보여 준다.

체계도 한 장의 기본 단위는 세부사업 한 개다. 1,401개 사업을 한 화면에 모두 펼치지 않고, 예산 계층에서 사업을 고른 뒤 선택한 사업의 상세 체계도를 그린다.

## 2. 자료별 역할

### 열린재정 `ExpenditureBudgetAdd2`

- 정본: 연도, 부처, 회계, 분야, 부문, 프로그램, 단위사업, 세부사업
- 정본 금액: `Y_YY_DFN_KCUR_AMT` 국회확정액
- 목·세목별 예산 구성과 규칙 기반 집행채널
- `direct` 채널은 다른 분류에 포함되지 않은 목·세목의 기본 분류일 수 있으므로 실제 직접 수행의 증거로 사용하지 않는다.

### 부처 세부사업 설명자료 PDF

- 사업 목적·내용
- 법적 근거와 추진 경위
- 시행방법, 시행주체, 수혜자, 지원율
- 예산 산출근거와 내역사업
- 사업 집행절차와 분기별 일정
- 성과지표, 결산, 외부 지적과 평가

각 사실에는 PDF 파일, 쪽, Kordoc 청크 범위와 근거 문장을 보존한다.

### 지방재정365 `QWGJK`

- 지자체명·코드와 광역/기초 구분
- 지방 세부사업명·코드
- 국비·시도비·시군구비, 편성액, 예산현액, 지출액
- 조회 기준일과 검색 키워드

중앙사업과 지방사업의 연결은 명칭 키워드 탐색 결과이므로 `keyword_candidate`다. 중앙사업에서 지방사업으로 확정 교부된 관계로 단정하지 않으며, 광역·기초 금액도 같은 재원을 중복 표현할 수 있어 중앙예산과 합산하거나 직접 대사하지 않는다.

선이 과도하게 겹치는 사업은 G5 체계도에 국비 금액순 최대 12개 후보 노드만 놓는다. 모든 LOFIN 후보 행과 금액은 삭제하지 않고 `예산·지역 흐름` 탭의 표에 보존한다.

## 3. 근거 강도

체계도는 다음 세 등급으로 구분한다.

1. `documented_flow`: PDF의 화살표, 번호 순서 또는 다행 추진계획 표에서 단계와 순서를 직접 확인한 사업
2. `structured_facts`: PDF에서 목적·근거·시행주체·수혜자·성과는 확인했지만 명시적 절차 순서가 부족한 사업
3. `api_only`: 열린재정의 예산 계층·금액·목·세목만 확보한 사업

화면의 G0~G6 단계는 여러 사업을 같은 방식으로 읽기 위한 **표시상 분류**다. 원문에 없는 승인, 반려, 보완, 회귀, 병목, 혁신, 담당기관 또는 순서를 자동 생성하지 않는다.

## 4. 표시 단계

| 단계 | 표시 내용 |
|---|---|
| G0 근거·목적 | 사업 목적, 법적 근거, 추진 경위 |
| G1 예산 확정·구성 | 국회확정액, 목·세목, 금액과 비중 |
| G2 시행 구조 | 담당부처, 시행방법·주체, 지원율, 수혜자 |
| G3 신청·선정·협약 | 원문에 명시된 신청, 심사, 선정, 협약, 교부 조건 |
| G4 집행·사업수행 | 명시적 집행절차, 일정, 실제 활동과 산출물 |
| G5 지방재정 반영 | LOFIN 광역·기초 지방사업 후보 |
| G6 성과·정산·환류 | 성과지표, 평가, 결산, 지적사항과 후속조치 |

단계 배치는 분석용 분류이며, 절차 순서로 보려면 각 노드와 연결의 근거 등급을 함께 확인해야 한다.

## 5. 노드와 연결

모든 노드는 사업 ID, 단계, 수행주체 레인, 행위명, 상세 설명, 금액, 근거 등급과 출처를 가진다.

연결은 다음을 구분한다.

- `sequence`: PDF에서 확인한 절차 순서
- `fund`: 합산 가능한 열린재정 목·세목 예산 구성
- `delivery`: 문서에 명시된 시행·지원 관계
- `information`: 보고, 통보, 제출 등 정보 전달
- `feedback`: 원문에 명시된 평가·보완·정산 환류
- `candidate`: LOFIN 키워드 후보 관계

추론 또는 표시를 위한 연결은 `derived`로 표시하고, 문서에서 직접 확인한 연결과 같은 강도로 그리지 않는다.

## 6. 품질 원칙

- 이름·계층·금액은 열린재정 API만 정본으로 사용한다.
- PDF 금액이나 LOFIN 금액으로 중앙예산 정본을 덮어쓰지 않는다.
- PDF 확정 매칭만 정규 사업에 붙이고 `ambiguous`와 `unmatched`는 별도 보존한다.
- 모든 노드와 연결 ID는 입력 자료만으로 결정적으로 생성한다.
- 한 사업의 예산 내역 합은 해당 사업의 국회확정액과 일치해야 한다.
- 출처가 없는 실제 절차나 행위자는 그리지 않고 `자료 미확보`로 남긴다.
- 인쇄물에도 기준연도, 출처, 근거 등급과 LOFIN 중복 위험을 남긴다.

핵심 원칙은 **보기는 원자료보다 자세하게 만들되, 주장은 원자료보다 강하게 만들지 않는 것**이다.
